  • User Story 001: Authenticating users
    • As Charity User
    • I want to authenticate
    • So that I can access my space
    Scenario 01: Authenticating on premise Charity Administrator
    • Given there are 3 Charity Administrators
    • And the user about to be authenticated matches the administrator
    • When the user is authenticated
    • Then the user should be authenticated on premise
    • And the user is redirected to the home page
    Scenario 02: Authenticating off premise Regular Charity User
    • Given the Charity Authentication type is off Premise
    • And the off premise authentication service authorises the user
    • When the user is authenticated
    • Then the user should be authenticated off premise
    • And the user is redirected to the home page
    Scenario 03: Failing to authenticate off premise unknown User
    • Given the Charity Authentication type is off Premise
    • And the on premise authentication service does not authorise the User
    • When the user is authenticated
    • Then the user should not be authenticated off premise
    • And there should be an error message for the user name
    • And the user stays on the login page
  • User Story 002: Charity Administration
    • As Charity Administrator
    • I want to be able to update the Charity authentication settings
    • So that I can manage the Charity off premise authentication
    Scenario 01: Prepopulating a Charity Administration details
    • Given an existing Charity has a name and a description
    • And the Charity has an off premise authentication type
    • When displaying the Charity Administration page
    • Then the page should be prepopulated with existing details
    Scenario 02: Persisting web site Url for Charity with off premise authentication
    • Given the Charity authentication type is off premise
    • And a valid web service url as been entered
    • When saving the web service url
    • Then the web service url should be persisted against the relevant Charity
    • And the execution result should be successful
    Scenario 03: Failing to persist web site Url
    • Given the Charity authentication type is off premise
    • And an invalid web service url as been entered
    • When saving the web service url
    • Then the web service url should be not be persisted
    • And the execution result should be unsuccessfull
  • User Story 003: Manage Administrators
    • As Charity Administrator
    • I want to Add, Edit, Delete Administrators
    • So that I can manage the Charity Administrators
    Scenario 01: View all Administrators
    • Given the Charity has 5 Administrators
    • When all Administrators are listed
    • Then there should be 5 Administrators
    • And each Administrator should have the corresponding UserName and fullName
    Scenario 02: Deleting an existing Administrator
    • Given the Charity has 3 Administrators
    • When Deleting the last Administrator
    • Then the relevant Administration should have been deleted
    • And there should be 2 Administrators remaining
